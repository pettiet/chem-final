# Chemistry Final · Study Hub

A mobile-first, single-user chemistry study site for the Spring chemistry final. Nine bite-sized modules (lessons + randomized quizzes), a cumulative 30-question exam with a personalized study plan, and progress / badges / streak tracking — all in pure HTML / CSS / vanilla JS. 100% static, deployable to GitHub Pages with zero build step.

## Test locally

The site is plain HTML/JS. Double-click `index.html` and it works.

For more accurate paths and to test link previews properly, serve it:

```bash
python3 -m http.server 8000
# open http://localhost:8000 on your phone or computer
```

In Chrome DevTools, switch to the iPhone 14 Pro device preview to see the intended mobile layout.

## Deploy to GitHub Pages

1. Create a public repo on GitHub (name it whatever you like — e.g. `chem-final`).
2. Push these files to the `main` branch:

   ```bash
   git init
   git add .
   git commit -m "Initial study site"
   git branch -M main
   git remote add origin https://github.com/<your-username>/<your-repo>.git
   git push -u origin main
   ```
3. On GitHub: **Settings → Pages → Source: `Deploy from a branch`**, **Branch: `main`**, **Folder: `/ (root)`**, then **Save**.
4. The site goes live at `https://<your-username>.github.io/<your-repo>/` in about 1–2 minutes.

Bookmark that URL on the iPhone home screen (Safari → Share → Add to Home Screen) for an app-like icon.

## Adding more questions later

All quiz content lives in **`js/questions.js`**. Each module's questions are an array of *generator functions* — each call produces a fresh question (with randomized numbers/compounds where applicable), which is what keeps repeat-quizzing from being memorization.

To add a new question:

1. Open `js/questions.js`.
2. Find the module's array (e.g. `Q.solutions = [...]`).
3. Append a new generator function returning a `{ type, prompt, ... }` object. Three shapes are supported: `mc` (multiple choice), `numeric` (with tolerance), and `text` (case-insensitive accepted alternates).
4. Commit and push. GitHub Pages picks it up on the next refresh.

Adding new lesson cards: edit **`js/lessons.js`** the same way — each module has an array of card objects with `title`, `body`, `example`, and an optional `widget` ID.

## Reset progress

Three ways:

- Press the **"Reset progress"** button at the bottom of the dashboard and confirm.
- **Long-press** that same button (~1.2s on iPhone) to reset without the confirm dialog.
- Open DevTools console and run `localStorage.clear()` — full nuke.

All progress lives in a single `localStorage` key (`chemFinal.progress`), so clearing it returns the site to a fresh state.

## Regenerate the link-preview image

The Open Graph preview (`og-image.png`) is generated by `make_og_image.py`. Edit the script and re-run:

```bash
python3 make_og_image.py
```

Requires Python 3 with Pillow (`pip3 install Pillow`).

## File layout

```
/
├── index.html                  # Dashboard
├── exam.html                   # Cumulative exam
├── cheat-sheet.html            # Printable cheat sheet
├── og-image.png                # Link-preview image
├── make_og_image.py            # Re-generates og-image.png
├── css/style.css               # All styles (mobile-first, dark)
├── js/app.js                   # Shared: progress, badges, streak, ring renderer
├── js/quiz-engine.js           # Lesson decks + quiz runner + interactive widgets
├── js/questions.js             # Question bank (generator functions)
├── js/lessons.js               # Lesson card content
├── modules/                    # 9 nearly-identical module pages
│   └── ...
└── README.md
```

No build step, no `npm install`, no frameworks. The only external dependency is `canvas-confetti` (loaded lazily from CDN, only when a mastery animation plays).
